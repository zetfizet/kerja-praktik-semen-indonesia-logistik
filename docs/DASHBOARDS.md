## Overview\n\nThree complementary analytics platforms provide different views of warehouse data:\n\n```\nMetabase (Business Intelligence)\n   → User-friendly SQL interface\n   → Pre-built dashboards & reports\n   → Best for: Business users, exploratory analysis\n\nGrafana (Monitoring & Dashboards)\n   → Real-time metrics & alerts\n   → Custom time-series visualizations\n   → Best for: Operations, performance monitoring\n\nSuperset (Data Exploration)\n   → Advanced analytics capabilities\n   → Semantic layer & ad-hoc queries\n   → Best for: Data analysts, complex queries\n```\n\n---\n\n## Metabase (Port 3000)\n\n### About Metabase\n\n**Purpose:** Business Intelligence & visual analytics  \n**Target Users:** Business analysts, managers, stakeholders  \n**Deployment:** Docker container  \n**License:** Open source (AGPL)\n\n### Starting Metabase\n\n```bash\nbash scripts/metabase/start_metabase.sh\n```\n\n**What it does:**\n1. Pulls Metabase image\n2. Starts container (port 3000)\n3. Initializes database\n4. Waits for web UI ready\n\n**Access:**\n```\nURL: http://localhost:3000\nInitial Setup Required: Yes (first time)\n```\n\n### First Time Setup\n\n#### Step 1: Create Admin Account\n\n1. Open http://localhost:3000\n2. Enter admin credentials:\n   - Email: admin@warehouse.local\n   - Password: (your choice, min 8 chars)\n   - Name: Admin\n3. Click \"Let's get started\"\n\n#### Step 2: Add Database Connection\n\n1. Click **Settings** (gear icon) → **Admin settings**\n2. Click **Databases** → **Add database**\n3. Configure warehouse connection:\n   ```\n   Database type:        PostgreSQL\n   Display name:         Warehouse\n   Host:                 localhost\n   Port:                 5433\n   Database name:        warehouse\n   Username:             postgres\n   Password:             postgres123\n   ```\n4. Click **Test connection**\n5. Click **Save**\n\n#### Step 3: Test Query\n\n1. Click **+ New** → **SQL query**\n2. Paste test query:\n   ```sql\n   SELECT COUNT(*) as total_drivers FROM public.drivers;\n   ```\n3. Click **Execute**\n4. Verify result shows driver count\n\n### Using Metabase\n\n#### Creating a Dashboard\n\n1. Click **+ New** → **Dashboard**\n2. Name it (e.g., \"Daily Operations\")\n3. Click **Create dashboard**\n4. Click **Edit** → **Save changes**\n\n#### Adding Visualizations\n\n1. In dashboard, click **+ Add a card**\n2. Click **New card**\n3. Select table (e.g., \"drivers\")\n4. Configure visualization:\n   ```\n   Visualization type: Table\n   Columns: name, status, hire_date\n   Rows: All\n   ```\n5. Click **Save**\n\n#### Running SQL Queries\n\n1. Click **+ New** → **SQL query**\n2. Write query (example):\n   ```sql\n   SELECT\n       DATE(start_time) as trip_date,\n       COUNT(*) as total_trips,\n       SUM(distance_km) as total_distance,\n       AVG(distance_km) as avg_trip\n   FROM public.perjalanan\n   WHERE start_time >= DATE '2026-04-01'\n   GROUP BY DATE(start_time)\n   ORDER BY trip_date DESC;\n   ```\n3. Click **Execute**\n4. Choose visualization (Table, Bar chart, etc.)\n5. Click **Save** → Select dashboard\n\n### Sample Queries for Metabase\n\n#### Dashboard: Driver Performance\n\n```sql\nSELECT\n    d.name,\n    COUNT(p.journey_id) as trips_month,\n    SUM(p.distance_km) as km_month,\n    AVG(p.distance_km) as avg_trip_km,\n    ROUND(100.0 * SUM(p.distance_km) / \n        (SELECT SUM(distance_km) FROM public.perjalanan \n         WHERE driver_id = d.driver_id AND \n         start_time >= DATE_TRUNC('month', NOW())), 2) as pct_total\nFROM public.drivers d\nLEFT JOIN public.perjalanan p ON d.driver_id = p.driver_id\n  AND p.start_time >= DATE_TRUNC('month', NOW())\nWHERE d.status = 'active'\nGROUP BY d.driver_id, d.name\nORDER BY trips_month DESC;\n```\n\n#### Dashboard: Weather Analytics\n\n```sql\nSELECT\n    location,\n    DATE(summary_date) as date,\n    temp_min,\n    temp_max,\n    temp_avg,\n    humidity_avg,\n    total_precipitation,\n    freshness_status\nFROM weather.daily_weather_summary dws\nJOIN weather.fact_weather_hourly fwh \n  ON dws.location = fwh.location\n  AND DATE(fwh.data_timestamp) = dws.summary_date\nWHERE summary_date >= CURRENT_DATE - 30\nORDER BY location, summary_date DESC;\n```\n\n### Stopping Metabase\n\n```bash\nbash scripts/metabase/stop_metabase.sh\n```\n\n---\n\n## Grafana (Port 3001)\n\n### About Grafana\n\n**Purpose:** Monitoring, metrics visualization, real-time dashboards  \n**Target Users:** Operations team, DevOps, system admins  \n**Deployment:** Docker container  \n**License:** Open source (AGPLv3) + commercial\n\n### Starting Grafana\n\n```bash\nbash scripts/grafana/start_grafana.sh\n```\n\n**Access:**\n```\nURL: http://localhost:3001\nDefault Username: admin\nDefault Password: admin\n```\n\n### First Time Setup\n\n#### Step 1: Change Default Password\n\n1. Open http://localhost:3001\n2. Login with admin/admin\n3. You'll be prompted to change password\n4. Enter new password (min 8 chars)\n5. Click **Save**\n\n#### Step 2: Add PostgreSQL Data Source\n\n1. Click **Configuration** (gear) → **Data Sources**\n2. Click **Add data source**\n3. Select **PostgreSQL**\n4. Configure:\n   ```\n   Name:              Warehouse\n   Host:              localhost:5433\n   Database:          warehouse\n   User:              postgres\n   Password:          postgres123\n   SSL Mode:          disable\n   ```\n5. Click **Test & Save**\n\n#### Step 3: Create Dashboard\n\n1. Click **+ Dashboard** (or **Dashboards** → **New Dashboard**)\n2. Click **Add a new panel**\n3. In query editor:\n   - Data source: \"Warehouse\"\n   - Write query (see examples below)\n4. Choose visualization\n5. Click **Save**\n\n### Sample Queries for Grafana\n\n#### Panel: Real-time Weather (Gauge)\n\n```sql\nSELECT\n    temperature_celsius as value,\n    location,\n    data_timestamp\nFROM weather.fact_weather_hourly\nWHERE location = 'Jakarta'\n  AND data_timestamp = (SELECT MAX(data_timestamp) \n                        FROM weather.fact_weather_hourly \n                        WHERE location = 'Jakarta')\nLIMIT 1;\n```\n\n#### Panel: Trip Volume Trend (Time Series)\n\n```sql\nSELECT\n    DATE_TRUNC('day', start_time) as time,\n    COUNT(*) as trips\nFROM public.perjalanan\nWHERE start_time >= NOW() - INTERVAL '30 days'\nGROUP BY DATE_TRUNC('day', start_time)\nORDER BY time ASC;\n```\n\n#### Panel: Weather Freshness Status (Status)\n\n```sql\nSELECT\n    CASE freshness_status\n        WHEN 'FRESH' THEN 2              -- Green\n        WHEN 'WARNING' THEN 1            -- Yellow\n        WHEN 'STALE' THEN 0              -- Red\n    END as status,\n    COUNT(*) as count\nFROM weather.fact_weather_hourly\nWHERE data_timestamp >= NOW() - INTERVAL '1 hour'\nGROUP BY freshness_status;\n```\n\n### Creating Alerts (Optional)\n\n```\n1. Configure alert conditions in panel\n2. Set threshold values\n3. Configure notification channel (email, webhook)\n4. Panel will alert when condition met\n```\n\n### Stopping Grafana\n\n```bash\nbash scripts/grafana/stop_grafana.sh\n```\n\n---\n\n## Apache Superset (Port 8088)\n\n### About Superset\n\n**Purpose:** Data exploration, visual analytics, semantic layer  \n**Target Users:** Data analysts, business intelligence teams  \n**Deployment:** Docker container  \n**License:** Open source (Apache 2.0)\n\n### Starting Superset\n\n```bash\nbash scripts/superset/start_superset.sh\n```\n\n**Access:**\n```\nURL: http://localhost:8088\nDefault Username: admin\nDefault Password: admin (change on first login)\n```\n\n### First Time Setup\n\n#### Step 1: Login & Change Password\n\n1. Open http://localhost:8088\n2. Login with admin/admin\n3. Click **Settings** (crown icon) → **Password**\n4. Enter new password\n5. Click **Save**\n\n#### Step 2: Add Database\n\n1. Click **Settings** → **Database connections**\n2. Click **+ Database**\n3. Fill in details:\n   ```\n   Database name:       warehouse\n   SQLAlchemy URI:      postgresql://postgres:postgres123@localhost:5433/warehouse\n   ```\n4. Click **Test Connection**\n5. Click **Save**\n\n#### Step 3: Create Chart\n\n1. Click **+ Chart**\n2. Select dataset: \"warehouse\" database\n3. Select table: (e.g., \"public.drivers\")\n4. Configure visualization:\n   - Visualization type: Table/Bar/Line/etc.\n   - Dimensions: Select columns\n   - Metrics: Count, Sum, Average, etc.\n5. Click **Update chart**\n6. Click **Save chart**\n\n### Using Superset\n\n#### Exploring Data (SQL Lab)\n\n1. Click **SQL** → **SQL Lab**\n2. Write query:\n   ```sql\n   SELECT * FROM public.drivers LIMIT 100;\n   ```\n3. Click **Execute**\n4. View results\n5. Click **Visualize** to create chart\n\n#### Creating Dashboards\n\n1. Click **+ Dashboard**\n2. Name it (e.g., \"Operations Dashboard\")\n3. Click **Create & Edit Dashboard**\n4. Click **+ Add Existing Chart**\n5. Select charts to add\n6. Arrange and resize\n7. Click **Save**\n\n#### Sample Exploration Query\n\n```sql\nSELECT\n    d.name,\n    d.status,\n    COUNT(p.journey_id) as total_trips,\n    SUM(p.distance_km) as total_distance,\n    MAX(p.start_time) as last_trip_date\nFROM public.drivers d\nLEFT JOIN public.perjalanan p ON d.driver_id = p.driver_id\nGROUP BY d.driver_id, d.name, d.status\nORDER BY total_trips DESC\nLIMIT 50;\n```\n\n### Stopping Superset\n\n```bash\nbash scripts/superset/stop_superset.sh\n```\n\n---\n\n## Dashboard Tool Comparison\n\n| Feature | Metabase | Grafana | Superset |\n|---------|----------|---------|----------|\n| **Ease of Use** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |\n| **SQL Capability** | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |\n| **Real-time Data** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |\n| **Alerting** | Manual | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |\n| **Visualization** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |\n| **Mobile Support** | ⭐⭐ | ⭐⭐⭐ | ⭐⭐ |\n| **Performance** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |\n| **Community** | Large | Very large | Growing |\n\n### Use Case Recommendations\n\n**Use Metabase if:**\n- ✅ Non-technical business users need dashboards\n- ✅ Want quick SQL queries without complexity\n- ✅ Need simple visualizations and reports\n- ✅ Small to medium team\n\n**Use Grafana if:**\n- ✅ Need real-time monitoring\n- ✅ Want sophisticated alerts\n- ✅ Monitoring system metrics & performance\n- ✅ DevOps/Infrastructure team\n\n**Use Superset if:**\n- ✅ Data analysts need advanced exploration\n- ✅ Complex queries and aggregations\n- ✅ Custom semantic layer\n- ✅ Enterprise analytics\n\n---\n\n## Troubleshooting Dashboard Tools\n\n### Metabase Connection Issues\n\n```bash\n# Check if container is running\ndocker ps | grep metabase\n\n# View logs\ndocker logs metabase\n\n# Restart container\ndocker-compose restart metabase\n```\n\n### Grafana Data Source Not Found\n\n1. Verify PostgreSQL is running: `docker ps | grep postgres`\n2. Verify connection parameters in Grafana UI\n3. Test connection from host:\n   ```bash\n   psql -h localhost -U postgres -d warehouse -p 5433\n   ```\n\n### Superset Slow Queries\n\n1. Check database performance:\n   ```sql\n   SELECT * FROM pg_stat_statements ORDER BY total_time DESC LIMIT 5;\n   ```\n2. Add indexes if needed\n3. Optimize query or materialized views\n\n---\n\n📖 **Next:** Read [Helper Scripts](SCRIPTS.md) for utility script reference  \n👈 **Back to:** [Main README](../README.md)\n
