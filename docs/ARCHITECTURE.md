## Component Details\n\n### 1. Data Source Layer\n\n#### DEVOM Warehouse (devom.silog.co.id)\n\n**Purpose:** Primary data source\n- 90+ business tables\n- Multiple schemas (activity, armada, delivery, driver, financial, master, support)\n- Live operational database\n\n**Connection Details:**\n```\nHost:           devom.silog.co.id\nPort:           5432 (PostgreSQL)\nDatabase:       om\nUser:           om\nPassword:       om (non-routable; requires dev env or VPN)\nProtocol:       TLS (standard PostgreSQL)\n```\n\n#### BMKG API\n\n**Purpose:** Real-time weather data\n- Indonesian Meteorological Agency official API\n- Updated every 2 hours\n- Multiple city coverage\n- Free and public\n\n**API Endpoints:**\n```\nhttps://api.bmkg.go.id/latest/weather\nhttps://api.bmkg.go.id/latest/forecast\n```\n\n---\n\n### 2. Orchestration Layer (Apache Airflow)\n\n#### Webserver (Port 8080)\n\n**Purpose:** Web UI and REST API\n- DAG visualization\n- Task execution monitoring\n- Manual DAG triggering\n- Log viewing\n\n**Components:**\n```\nFramework:      Flask (Python)\nComponents:     UI, REST API, WebSocket\nAuth:           Basic auth (admin/rafie123)\nVolume backup:  N/A (stateless)\n```\n\n#### Scheduler\n\n**Purpose:** DAG scheduling and triggering\n- Parses DAGs (every 30 seconds)\n- Maintains scheduler loop\n- Triggers DAG runs based on cron schedules\n- Manages task dependencies\n\n**Scheduling Logic:**\n```\nCheck time вҶ’ Is schedule match? вҶ’ Create DagRun\n     вҶ“           вҶ“              вҶ“\nEvery 30s    daily_warehouse_sync  (00:00 UTC)\n             weather_data_fetch    (every hour)\n```\n\n#### Executor (CeleryExecutor)\n\n**Purpose:** Distributed task execution\n- Sends tasks to worker queue\n- Monitors task completion\n- Collects results\n\n**Configuration:**\n```\nType:             CeleryExecutor\nBroker:           Redis (valkey:6379)\nResult Backend:   PostgreSQL\nWorkers:          1 (development setup)\nParallel Tasks:   Limited by worker count\n```\n\n#### Database (PostgreSQL, port 5432)\n\n**Purpose:** Airflow metadata storage\n- DAG run history\n- Task instance records\n- Connection definitions\n- Logs\n\n**Tables:**\n```\ndag               (DAG metadata)\ndag_run           (DAG execution records)\ntask_instance     (Individual task runs)\nlog               (Execution logs)\nconnection        (DB/API connections)\nvariable          (Global variables)\n```\n\n### 3. Message Broker (Redis/Valkey)\n\n**Purpose:** Celery task queue\n- Task distribution\n- Worker communication\n- Result passing\n\n**Configuration:**\n```\nImage:           valkey:9 (Redis compatible)\nPort:            6379\nStorage:         In-memory (ephemeral)\nPersistence:     Disabled (appendonly: no)\nURIs:\n  Broker:        redis://valkey:6379/0\n  Result Backend: db+postgresql://...\n```\n\n---\n\n## Data Flow\n\n### Daily Warehouse Sync Flow\n\n```\n[Scheduler] - Checks every 30s\n    вҶ“\n[00:00 UTC] - Schedule matches daily_warehouse_sync\n    вҶ“\n[Create DagRun] - New execution instance\n    вҶ“\n[Task 1: check_weather_data]\n  в”ңв”Җ Connect to devom.silog.co.id\n  в”ңв”Җ Query latest weather table\n  в”ңв”Җ Calculate freshness ratio\n  в””в”Җ Return status вҶ’ Success\n    вҶ“\n[Task 2: sync_warehouse_tables]\n  в”ңв”Җ Connect to source (om/om credentials)\n  в”ңв”Җ List all tables (SELECT * FROM information_schema.tables)\n  в”ңв”Җ For each of 90+ tables:\n  в”Ӯ  в”ңв”Җ fetch_all_rows (full scan)\n  в”Ӯ  в”ңв”Җ deduplicate (group by & distinct)\n  в”Ӯ  в”ңв”Җ batch_insert (5000 rows/batch)\n  в”Ӯ  в””в”Җ log_status (success/error)\n  в””в”Җ Return summary вҶ’ Success\n    вҶ“\n[Task 3: verify_warehouse_data]\n  в”ңв”Җ Query row counts in warehouse\n  в”ңв”Җ Compare with source counts\n  в”ңв”Җ Validate schema matches\n  в””в”Җ Write verification report\n    вҶ“\n[DagRun Complete]\n  в”ңв”Җ Duration: 5-15 minutes\n  в”ңв”Җ Status: Success/Failed\n  в””в”Җ Next run: Tomorrow 00:00 UTC\n```\n\n### Weather Data Fetch Flow\n\n```\n[Scheduler] - Every hour\n    вҶ“\n[00:00, 01:00, ..., 23:00 UTC] - Schedule matches\n    вҶ“\n[Create DagRun]\n    вҶ“\n[Task 1: fetch_weather_data]\n  в”ңв”Җ For each location (Jakarta, Bandung, ...)\n  в”Ӯ  в”ңв”Җ Make HTTP GET to BMKG API\n  в”Ӯ  в”ңв”Җ Parse JSON response\n  в”Ӯ  в”ңв”Җ Extract: temp, humidity, wind, precip\n  в”Ӯ  в””в”Җ Normalize timestamp to Asia/Jakarta TZ\n  в””в”Җ Collect all records in memory\n    вҶ“\n[Task 2: deduplicate_data]\n  в”ңв”Җ Query existing records (location + hour)\n  в”ңв”Җ Hash compare new vs existing\n  в”ңв”Җ If identical: skip\n  в”ңв”Җ If new: insert with ON CONFLICT DO NOTHING\n  в””в”Җ Count inserted: N records\n    вҶ“\n[Task 3: track_freshness]\n  в”ңв”Җ Calculate age of latest record\n  в”ңв”Җ Classify: FRESH (вүӨ60m) / WARNING (60-180m) / STALE (>180m)\n  в”ңв”Җ Store classification in metadata table\n  в””в”Җ Log metrics\n    вҶ“\n[Task 4: cleanup_old_data]\n  в”ңв”Җ Delete records > 30 days old\n  в”ңв”Җ VACUUM table (reclaim space)\n  в””в”Җ Log: \"Deleted N rows\"\n    вҶ“\n[DagRun Complete]\n  в”ңв”Җ Duration: 1-5 minutes\n  в”ңв”Җ Next run: Next hour\n```\n\n---\n\n## Integration Points\n\n### Airflow
